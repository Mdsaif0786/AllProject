Q 3. What is eventloop?

Ans =>

Event loop is an endless loop, which waits for tasks, executes them and then sleeps until it receives more tasks. The event loop executes tasks from the event queue only when the call stack is empty i.e. there is no ongoing task. The event loop allows us to use callbacks and promises.

Ans=>

A callback function is a function passed into another function as an argument, which is then invoked inside the outer function to complete some kind of routine or action. Here is a quick example: function greeting(name) { alert(`Hello, ${name}`); } function processUserInput(callback) { const name = prompt("Please enter your name."); callback(name); } processUserInput(greeting);
